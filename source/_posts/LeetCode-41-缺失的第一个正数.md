# LeetCode 41 - 缺失的第一个正数

## 题目描述

给你一个未排序的整数数组 `nums`，请你找出其中没有出现的最小的正整数。

**示例 1：**
```
输入：nums = [1,2,0]
输出：3
```

**示例 2：**
```
输入：nums = [3,4,-1,1]
输出：2
```

**示例 3：**
```
输入：nums = [7,8,9,11,12]
输出：1
```

**要求：** 时间复杂度 O(n)，空间复杂度 O(1)。

## 解题思路

题目要求 O(n) 时间和 O(1) 空间，意味着不能使用额外哈希表，也不能排序。

### 核心思想

利用**数组本身作为哈希表**。

长度为 n 的数组，缺失的第一个正数一定在范围 `[1, n+1]` 内。因此，我们可以将每个数放到它应该在的位置上：

- 数字 `x` 应该放在下标 `x-1` 处
- 例如：`1` 放在 `nums[0]`，`2` 放在 `nums[1]`，依此类推

### 算法步骤

1. 遍历数组，对于每个位置 `i`：
   - 如果 `nums[i]` 在 `[1, n]` 范围内，且 `nums[i]` 不在它应该在的位置（`nums[i] != i+1`），且目标位置上的数不是它自己（避免死循环），则**交换**
   - 否则 `i++`，继续处理下一个
2. 再次遍历数组，第一个 `nums[i] != i+1` 的位置，返回 `i+1`
3. 如果全部都对得上，返回 `n+1`

### 示例

以 `[3,4,-1,1]` 为例：
```
初始:  [3,  4, -1,  1]
        ↓   ↓   ↓   ↓
      位置1 位置2 忽略 位置?

第1步: [-1, 4,  3,  1]  (3 放到下标2)
第2步: [-1, 1,  3,  4]  (4 放到下标3)
第3步: [1, -1,  3,  4]  (1 放到下标0)
第4步: 扫描 → nums[1] = -1 ≠ 2，返回 2
```

## 代码

```cpp
//
// Created by cam xu on 2021/5/16.
//

#include <vector>

using namespace std;

namespace ns41 {

    class Solution {
    public:
        int firstMissingPositive(vector<int> &nums) {
            for (int i = 0; i < nums.size();) {
                if (nums[i] > 0 && nums[i] <= nums.size()
                    && nums[i] != i + 1 && nums[nums[i] - 1] != nums[i]) {
                    swap(nums[i], nums[nums[i]-1]);
                } else {
                    ++i;
                }
            }

            for (int i = 0; i < nums.size(); ++i) {
                if (nums[i] != i + 1) {
                    return i + 1;
                }
            }

            return nums.size();
        }

    private:
        void swap(int &a,int &b) {
            int c = a;
            a = b;
            b = c;
        }
    };

};
```

## 复杂度分析

- **时间复杂度**：O(n)，每个元素最多被交换一次
- **空间复杂度**：O(1)，原地交换，无额外空间

## 总结

这道题的经典之处在于利用数组下标作为哈希表的键，将数字放到它"该在的位置"。关键在于**只在能正确放置时才交换**，否则会陷入死循环。

同类题型还有 LeetCode 442（数组中重复的数据）、LeetCode 448（找到所有数组中消失的数字），都是同一思路。
